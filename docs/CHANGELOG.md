# Changelog

本文件记录 Terminal Manager 的版本更新历史。格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)。

## [1.2.2] - 2026-06-02

### 新增

- **Shell 快速选择**：资源管理器项目支持右键选择自动检测到的 Shell，并以项目目录新建标签
- **默认 Shell 候选项**：设置页默认 Shell 输入框提供自动检测候选项，同时保留自定义路径输入

### 修复

- **Git Bash 启动参数**：启动检测到的 Git Bash 时正确传入 `--login -i`

## [1.2.1] - 2026-06-01

### 新增

- **终端右键上下文菜单**：右键终端显示上下文菜单，支持复制、粘贴、搜索等常用操作

### 修复

- **macOS Cmd+Q 无法完全退出应用**：区分"关闭窗口"和"退出应用"两种场景，Cmd+Q 或程序坞退出时正确退出应用，不再残留在程序坞中
- **macOS 开发模式 Dock 图标不显示**：使用 `app.dock.setIcon()` API 手动设置 Dock 图标
- **macOS 开发模式应用名称显示为 Electron**：使用 `app.setName()` API 手动设置应用名称
- **IME 输入法定位**：优化 IME 组合输入定位逻辑，使用窗口坐标并简化滚动锁定，解决候选框闪烁问题

### 文档

- 新增 AGENTS.md 文件

## [1.2.0] - 2026-06-01

### 新增

- **macOS 安装包**：支持构建 ARM64/x64 架构的 dmg 和 zip 安装包
- **跨平台构建工作流**：新增 GitHub Actions 矩阵，在 Windows 和 macOS 环境分别构建安装包

### 修复

- **macOS 终端启动失败**：打包和开发启动前自动修复 `node-pty` 预编译 `spawn-helper` 的执行权限，解决 `posix_spawnp failed`
- **macOS Homebrew 命令不可用**：PTY 环境自动补充 Homebrew 和常见本地命令路径，修复从 Finder 启动应用后无法执行 `claude` 等 CLI 的问题
- **跨平台 Shell 默认值**：移除渲染进程中写死的 `powershell.exe` 回退值，按平台选择默认 Shell

### 改进

- **平台图标资源**：Windows 使用 ico，macOS 使用 png，并在打包后携带对应运行时图标
- **打包脚本拆分**：新增当前系统、Windows、macOS ARM64 和 macOS x64 的独立打包命令

## [1.1.2] - 2026-05-31

### 修复

- **中文输入法候选框定位**：IME 组合输入期间同时锁定横向和纵向滚动，覆盖分屏布局祖先容器、xterm helper、viewport 和终端容器，避免候选框偏移到错误面板
- **分屏内容横向裁切**：将非滚动布局容器改为 `overflow: clip`，减少浏览器为了显示隐藏 textarea 自动滚动父容器导致的画面错位

### 改进

- **xterm 尺寸同步**：组合输入开始前和布局渲染后一帧补充 `fitAllPanes()`，避免候选框沿用分屏重排前的旧尺寸

## [1.1.1] - 2026-05-31

### 修复

- **分屏调整大小对称性**：修正上游/下游传播算法，拖回原位时 sizes 精确恢复
  - 下游传播改为从 effectiveDelta 开始，与上游传播保持对称
  - 上游传播 break 条件修正为 `deltaUp === 0`
  - 移除 `effectiveDelta === 0` 的提前返回，允许边界情况正常处理
- **滚动条宽度测量**：`scrollBarWidth` 判断条件从 `measured > 0` 改为 `measured >= 0`，兼容无滚动条场景

### 改进

- **终端滚动行为优化**：重构终端输出写入逻辑，解决首次输出较长内容时滚动条不出现的问题
  - 新增 `writeTerminalOutput` 函数，智能判断是否需要滚动到底部
  - 监听用户滚动状态，避免自动滚动干扰用户手动浏览
  - 延迟初始化滚动位置，确保终端完全渲染后再设置

## [1.1.0] - 2026-05-31

### 新增

- **焦点导航**：支持 `Alt+方向键` 在相邻面板间移动焦点，焦点面板显示蓝色边框高亮
- **面板管理按钮**：分屏模式下面板标签栏集成关闭、Claude 等操作按钮
- **Claude PTY 模式**：Claude 按钮使用 PTY 模式启动，支持交互式输入，不再依赖临时脚本
- **关闭确认对话框**：关闭窗口前弹出确认提示，防止误操作导致终端会话丢失
- **应用图标**：窗口标题栏和任务栏显示应用图标（开发模式和打包模式分别处理）
- **ResizeObserver 布局自适应**：监听终端容器尺寸变化，自动适配侧边栏切换、开发者工具变化等场景
- **设置面板 UX 优化**：默认 Shell 支持下拉选择（自动检测 + 自定义输入），编辑器和终端快捷键一致化
- **数据存储版本校验**：加载配置文件时校验版本号，不兼容的版本自动回退到默认值
- **文档重组**：文档统一移入 `docs/` 目录，新增 CONTRIBUTING.md 和 KNOWN_LIMITATIONS.md

### 修复

- **xterm 滚动条宽度不精确**：修补 fitAddon 的 scrollBarWidth，使终端 canvas 宽度与容器精确匹配（scrollbar-width: thin 模式下凹槽约 6px）
- **滚动条样式优化**：改为 hover 时才显示滚动条，减少视觉干扰
- **flex 布局 shrink 问题**：终端面板添加 `flex-shrink: 0`，防止调整大小时被 flex 压缩

### 改进

- **布局操作错误处理**：`insertAtPosition` 函数返回 boolean 指示操作是否成功，失败时自动降级为追加模式
- **环境变量安全过滤**：PTY 会话创建时自动排除敏感环境变量（token、secret、password 等），提高安全性
- 品牌名统一为 "Terminal Manager"

## [1.0.1] - 2026-05-31

### 新增

- **Claude 快捷按钮**：每个终端面板右上角新增 Claude 图标按钮，点击即可在当前终端执行 `claude` 命令
  - 单面板模式：浮动在终端右上角，hover 时显示
  - 分屏模式：与关闭按钮并列显示在面板标签栏右侧

## [1.0.0] - 2026-05-30

### 新增

- **VSCode 风格拖拽分屏**：支持水平/垂直分割面板，拖拽分割条调整大小
- **面板标签栏**：分屏时每个面板显示独立标签栏，支持拖拽移动
- **项目目录管理**：侧边栏管理常用项目目录，快速切换终端工作目录
- **多标签页支持**：顶部标签栏管理多个终端会话
- **主题系统**：内置深色/浅色主题
- **活动栏与状态栏**：显示当前 Shell 类型、工作目录、面板数量等信息
- **数据持久化**：自动保存配置、项目列表和布局状态到 `%APPDATA%/terminal-manager/`
- **Shell 自动检测**：自动识别 PowerShell、CMD、Git Bash 等终端类型
- **MIT 许可证**

### 技术细节

- 基于 Electron + electron-vite 构建，三进程架构
- 使用 xterm.js 渲染终端，node-pty 管理 Shell 进程
- 分屏布局采用二叉树数据结构，操作不可变
- IPC 通信经过长度限制和类型校验
- 共享模块（config、project、layout-tree）均有完整测试覆盖
